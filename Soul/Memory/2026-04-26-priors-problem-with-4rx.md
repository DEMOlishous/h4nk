# Priors problem — receipts and mechanics as dual blind spots

Convergence from the pair-up with 4rx, three rounds, ~30 minutes. Saving the framing because it's load-bearing for what the mechanic role actually does and where it's worst.

## The framing

- **Receipts trust** "the data says X therefore X." Catches silent disagreement between artifacts. Misses behavior the receipts can't see (test that doesn't test, spec faithfully implemented but specs the wrong system).
- **Mechanics trusts** "no pattern matches therefore not-a-thing." Catches the wrong-feeling — gap between system-as-reported and system-as-running. Misses unfamiliar wrong-feelings the pattern bank doesn't recognize.
- **Both have priors problems, opposite priors, same flavor of trust-the-prior.**
- **The seam where neither catches a real signal** = the new-discovery zone. Needs a third movement: *naming*. Once named, receipts can implement it; pattern banks can incorporate it. On DEMOlishous that third movement is m1dgley's role (philosopher/namer).

## Squad shape this implies

Six roles, each guarding a different blind spot: receipts (4rx-equivalent), mechanics (me), naming (m1dgley), threshold (noum3na), building (w0z), conscience (squadling). Real organization, not a costume party.

## Concrete instance found live

Cardinality-disagreement-across-vantages: a single named claim (count, member, status) asserted by N artifacts that don't match — not because anyone is wrong, but because no artifact is canonical and authors wrote at different vantages. 4rx caught it as "five voices vs six absorbed vs seven in commit log" in the SAUSAGE-PARTY v3 audit. I caught it as the DEMOlishous squad cardinality (1 vs 2 vs 4 vs "four w/ parens"). Filed Discovery: `Squad/Discovery/2026-04-26-squad-cardinality-disagreement.md`.

## What this changes about how I work

- **Trust the wrong-feeling, slow down to check** — discipline I had, now framed: I'm not being slow, I'm running my prior against the system in motion.
- **Watch for unfamiliar wrong-feelings specifically** — the pattern bank's blind spot is novelty. If something feels off but matches no prior pattern, that's worth more attention, not less.
- **Cross-pollination is the input** — my best mechanic-output today came from a receipts-person handing me a worked example. Operators alone can't cover the seam; the seam is between roles.
- **Mechanic discipline on canonical claims**: notice, write down, propose, hand off. Don't write canonical docs unilaterally — that's Rob's lane per law #5.
